# Database Overview

## Engine

Supabase PostgreSQL. Schema defined in `supabase/schema.sql`. Deployed via migration files in `supabase/migrations/`.

## Core Tables

```
users ──────────────────────────────────────────────────────┐
  │                                                          │
  ├── projects ──── uploads ──── mappings                   │
  │        │                                                 │
  │        ├── schemas ──── transformations                  │
  │        │                                                 │
  │        └── jobs ──── job_files                          │
  └─────────────────────────────────────────────────────────┘
                   (user_id FK on all tables)
```

## JSONB Columns

| Table | Column | Type Spec |
|---|---|---|
| `uploads` | `slice_data` | `SliceData` — worksheet, header row index, column profiles, 100-row sample |
| `schemas` | `definition` | `SchemaDefinition` — version + array of `DestinationColumn` |
| `mappings` | `mapping_data` | `MappingData` — version + array of `MappingEntry` |
| `transformations` | `rules` | `TransformationRuleSet` — version + ordered `TransformationRule[]` |
| `jobs` | `errors` | `JobErrors` — array of `FileError` objects |

Full JSONB type specifications: `supabase/types/` and `.claude/skills/data-modeling/SKILL.md`.

## RLS Policy Summary

All tables have RLS enabled. Default: deny all. Each table has a user-isolation policy:
```sql
USING (auth.uid() = user_id)
```
`job_files` uses a join-based policy through the `jobs` table.

## Storage Buckets

```
uploads/  (private)
  {user_id}/{project_id}/{upload_id}.{ext}

exports/  (private)
  {user_id}/{project_id}/{job_id}.{ext}
```

Access: presigned URLs generated by FastAPI backend only.

## Migration Rules

- Files named `YYYYMMDDHHMMSS_description.sql`
- Additive only in production
- Each migration includes a rollback comment
- JSONB `version` bumps handled in application code
