## рҹ“Ӣ API Endpoints

### Authentication API

```javascript
// Login (public)
login(username, password)
  вҶ’ Returns: { session_token, user }

// Logout (authenticated)
logout(sessionToken)
  вҶ’ Returns: { success }

// Validate session
validateSession(sessionToken)
  вҶ’ Returns: { valid: true/false }

// Get current user
getCurrentUser(sessionToken)
  вҶ’ Returns: { username, email, emp_id, emp_name, role, permission }
```

### Shift API (all require sessionToken)

```javascript
getCurrentShift(sessionToken)
startShift(sessionToken)
endShift(sessionToken)
getRecoverableShift(sessionToken)
resumeLastShift(sessionToken)
```

### Break & Lunch API (all require sessionToken)

```javascript
startBreak(sessionToken)
endBreak(sessionToken)
startLunch(sessionToken)
endLunch(sessionToken)
```

### Task & Activity API (all require sessionToken)

```javascript
logTask(sessionToken, description)
```

## рҹӣ‘ Common Mistakes to Avoid

| вқҢ Mistake | вң… Solution |
|-----------|-----------|
| Not updating function signatures to include sessionToken | Add `sessionToken` parameter to all backend functions |
| Storing plaintext passwords | Always use `CryptoService.hashPassword()` |
| Forgetting to pass sessionToken from frontend | Every API call must pass `_sessionToken` |
| Old `Session.getActiveUser().getEmail()` still in code | Replace with `ctx = AuthService.getCallerContext(sessionToken)` |
| Not clearing localStorage on logout | `localStorage.removeItem("ATS_SESSION")` |
| Incorrect Users sheet schema | Follow exact column order (Username, Hash, Salt, Token...) |
| Missing Constants.gs file | Must be deployed before any other service uses it |
| Missing CryptoService.gs file | Required for password hashing |

## рҹ“Ҳ Performance Notes

### Session Token Lookup
- First call: Sheet query (slow)
- Subsequent calls: 30-minute cache (fast)
- Cache key: `ats:user:{sessionToken}`

### Stale State Detection
- Checks on every shift action
- Runs every 30s in background (after fetch)
- Auto-closes shifts > 12 hours (configurable)
- Auto-closes activities > 4 hours (configurable)

### Optimization
- Use `getRange(row, col, numRows, numCols)` instead of `getDataRange()`
- Cache active shift during session
- Clear cache on logout
